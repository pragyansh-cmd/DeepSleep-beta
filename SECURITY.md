# Security Policy

If you discover a security issue in DeepSleep, please do not open a public issue with exploit details first.

Instead:

1. Open a private GitHub security advisory if available.
2. If that is not available, contact the maintainer directly through GitHub.
3. Include reproduction steps, impact, and affected versions.

## Scope

Please report:

- arbitrary command execution issues
- unsafe file access behavior
- secrets exposure
- dependency vulnerabilities with real impact

## Response

Best effort:

- acknowledge the report quickly
- confirm severity and reproduction
- ship a fix or mitigation
- publish a coordinated changelog note when appropriate
